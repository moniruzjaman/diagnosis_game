# ধানের ডাক্তার সিমুলেশন (Smart Farmer Simulation)

An engaging agricultural simulation game for rice disease identification and proper management using modern technology.

## ✨ Features

- 🌾 **Interactive Simulation**: Learn to identify rice diseases through hands-on field observation
- 🧠 **Knowledge-Based Gameplay**: Practice the CABI (Centre for Agriculture and Bioscience International) methodology
- 🏆 **Achievements & Scoring**: Earn points and unlock achievements as you progress
- 📚 **Encyclopedia**: Access a comprehensive knowledge base of agricultural terms and practices
- 🎮 **Multiple Difficulty Levels**: Easy, Medium, and Hard modes for all skill levels
- 🔊 **Sound Effects**: Immersive audio feedback for actions and achievements
- 🤖 **AI-Powered Hints** (Vercel deployment): Dynamic, contextual hints generated by AI

## 🚀 Deployment Options

### Option 1: Deploy to Vercel (Recommended)

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/moniruzjaman/diagnosis_game)

**Vercel deployment includes AI-powered features:**
- Dynamic hint generation based on game context
- AI Q&A assistant for agricultural questions

1. Click the button above or go to [Vercel](https://vercel.com)
2. Import the repository: `https://github.com/moniruzjaman/diagnosis_game`
3. Vercel will automatically detect the Vite framework
4. Click Deploy

The project includes:
- `vercel.json` - Configuration for optimal deployment
- `api/` - Serverless functions for AI features

### Option 2: Deploy to GitHub Pages (Static)

GitHub Pages deployment works as a static site without AI features.

1. Push the code to your GitHub repository
2. Go to your repository **Settings** → **Pages**
3. Under "Build and deployment" → Source, select **"GitHub Actions"**
4. The repository includes `.github/workflows/deploy.yml` workflow
5. Your site will be available at: `https://[username].github.io/diagnosis_game/`

## 🏃 Run Locally

**Prerequisites:** Node.js 18+

1. Clone the repository:
   ```bash
   git clone https://github.com/moniruzjaman/diagnosis_game.git
   cd diagnosis_game
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Run the development server:
   ```bash
   npm run dev
   ```

4. Open http://localhost:3000 in your browser

**Note:** For AI features locally, use `vercel dev` command:
```bash
npm install -g vercel
vercel dev
```

## 🔨 Build for Production

```bash
npm run build
```

The production build will be created in the `dist` folder.

## 🎮 Game Phases

The simulation follows the CABI methodology:

1. **Observe (মাঠ পর্যবেক্ষণ)**: Field scouting and symptom observation
2. **Diagnose (রোগ নির্ণয়)**: Problem identification and cause analysis
3. **Measure (পরিমাপ)**: Damage assessment and economic threshold level (ETL)
4. **Think (বিশ্লেষণ)**: Risk and context evaluation
5. **Act (ব্যবস্থাপনা)**: IPM (Integrated Pest Management) decision making
